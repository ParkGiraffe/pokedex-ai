# 프로젝트 하드룰

> 이 프로젝트의 모든 산출물(코드·문서·UI 텍스트·AI 응답)에 예외 없이 적용된다.

## 표현 규칙

- **이모지 금지**. 코드·문서·UI·커밋 메시지 어디에도 한 글자도 쓰지 않는다.
- **고유명사 왜곡 금지**. 행사명·브랜드명·기술명·포켓몬 이름은 공식 한국어 표기 그대로 사용한다 (예: "스칼렛/바이올렛"을 "SV"로 줄이는 건 OK, "스카바"로 변형 금지).
- **한국어 톤 우선**. 문서 본문도 한국어로 작성한다. 영문은 코드 식별자·외부 라이브러리명·고유명사에 한정.
- **영어 직역 금지**. "Major threat" → "메이저 위협" 같은 직역 대신 한국 SV 커뮤니티에서 실제 통용되는 어휘를 쓴다. `lexicon.md` 참고.

## 작업 규칙

- 작업 시작 전 이 파일과 `code-principles.md`, `client-stack.md`, `lexicon.md`, 그리고 `docs/specs/` 아래 최신 스펙을 모두 읽어 컨텍스트를 복원한다.
- 모든 모델 호출은 Opus를 사용한다.
- 새 패키지를 도입할 때는 사용자 승인을 먼저 받는다 (p2z 원칙).

## 커밋 규칙

- `Co-Authored-By: Claude ...` 트레일러를 절대 추가하지 않는다. 사용자 본인 이름만 author로 들어간다.
- 커밋 메시지에 이모지 금지.

## AI 응답 규칙 (Claude Code 대화형 분석)

- 사용자가 web app에서 export한 JSON을 paste하면, 그 JSON을 1차 진실로 신뢰하고 분석한다.
- 응답에 한국 SV 커뮤니티 어휘를 사용한다 (`lexicon.md`).
- 추천은 근거(상성·수치·메타)와 함께 제시한다. "그냥 좋아서"식 답변 금지.
- 확률을 말할 때는 데미지 계산·승률 추정 모두 결정론적 공식(`pokedex-core/formula`)을 기준으로 한 값임을 명시한다.
