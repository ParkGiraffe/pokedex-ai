# 포켓몬 챔피언스 분석 도구 개발기 (시리즈 개요)

개인 프로젝트 `pokedex-agent` 개발 과정을 정리한 시리즈다. 포켓몬 챔피언스 싱글배틀을
분석하는 웹앱과, 그 위에 얹은 Claude 기반 AI 분석 기능을 만든 기록이다. 중립 톤으로
쓰고, 겪은 버그와 해결 과정을 빠뜨리지 않는다.

작성 기반: git 커밋 137개, 프로젝트 메모리, 개발 세션 로그.

## 편 구성

1. **왜 만들었나, 그리고 도메인 파운데이션** — 챔피언스로의 타깃 피벗, 모노레포 골격,
   결정론 도메인 라이브러리(pokedex-core), PokeAPI 데이터 수집, 데미지·실수치·스피드 공식
2. **웹앱 네 페이지** — React + Vite + TanStack으로 만든 계산기·스피드·도감·파티빌더
3. **AI 분석 도입과 시행착오** — 클립보드 paste 왕복 UX를 폐기하고 서버가 Anthropic을
   직접 호출하기까지, 한국어 고유명사 fabricate 문제와 2패스 보정
4. **백엔드 전환: Fastify에서 NestJS로** — MikroORM·Postgres 도입, 워크스페이스 패키지
   번들 함정, 내부 로그인·JWT·일일 쿼터
5. **리팩토링과 품질** — 브라우저 번들에서 새던 서버 전용 데이터 격리, lint 게이트 도입,
   메가스톤 표기 오류 교정
6. **커뮤니티·AI 확장** — 공유 링크, 리더보드, 전적·코칭, 노력치 역산기, 매치업 매트릭스,
   리빙 메타, 배틀 스크린샷 조언
7. **겪은 버그와 교훈, 그리고 회고** — 반복해서 발목 잡은 함정들과 다음 계획

## 스크린샷 정책

현재 동작하는 기능은 실제 화면을 캡처해 싣는다. 이미 사라진 과거 UI(초기 클립보드 플로우 등)는
코드 스니펫과 아키텍처 설명으로 대체한다. 옛 버전은 데이터 포맷·의존성이 달라 그대로 재현하기
어렵기 때문이다.
