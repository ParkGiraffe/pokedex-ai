# 5편 — 리팩토링과 품질

기능이 쌓이면 어딘가 삐걱대기 시작한다. 데이터가 중복 관리되는 것 같고, 쓸데없는 파일이
보이고, 자원이 일원화되지 않은 느낌. 이 편은 그 감을 증거로 확인하고 손본 기록이다.

## 서버 전용 데이터가 브라우저로 새고 있었다

가장 큰 발견은 번들에 있었다. `pokedex-core`의 배럴 export가 gen9 Smogon 폴백 데이터를 함께
내보내고 있었다. 이 데이터(약 2MB)는 챔피언스 공개 샘플이 없는 포켓몬의 예상 셋을 채우는
용도로, 오직 서버측 `battle-engine`만 쓴다. 그런데 배럴 export 때문에 브라우저 번들에도 그대로
인라인되고 있었다. 브라우저가 쓰지도 않는 2MB를 매번 받고 있던 셈이다.

해결은 데이터의 소비 경로를 분리하는 것이었다. gen9 폴백 함수(`smogonSets` 등)를 별도 모듈로
빼고, node 전용 서브패스(`@pokedex-agent/pokedex-core/fallback`)로만 노출했다. 메인 배럴에는
브라우저 안전 유틸만 남겼다. 결과적으로 **클라이언트 메인 번들이 3.1MB에서 2.4MB로** 줄었다.

여기서 판단이 하나 필요했다. 폴백 자체를 지울까? 측정해보니 챔피언스 공개 샘플은 합법 로스터
274종 중 169종(62%)만 덮었다. 나머지 38%는 이 폴백에 의존하고 있었다. 그래서 폴백은 유지하되
브라우저에서만 격리하는 쪽으로 정했다. 지우는 게 아니라 옮기는 게 답이었다.

## lint이 사실 아무것도 안 하고 있었다

코드 컨벤션을 따른다고 믿고 있었는데, 확인해보니 lint 설정 파일이 하나도 없었다. `pnpm lint`는
`turbo lint`를 부르고, 그 turbo lint는 아무 패키지에도 lint 스크립트가 없어서 그냥 통과하고
있었다. 게이트가 텅 비어 있던 것이다.

eslint와 prettier를 도입했다. 사내 컨벤션을 따르되, 패키지마다 설정을 복붙하는 대신 루트에
공유 base 설정을 두고 각 패키지가 얇게 확장하는 형태로 일원화했다. 룰 강도는 점진적으로 잡았다.
import 정렬·미사용 import·포맷팅은 안전하게 자동 수정되니 error로, 기존 코드 위반이 많은
타입 안전 계열은 우선 warn으로 두어, 한 번에 녹색 게이트를 만들고 나서 점차 조이기로 했다.

(이 자동 수정에는 함정이 하나 있었는데, 그건 7편에서 다룬다.)

## 메가스톤 이름이 틀어져 있었다

데이터 정합성을 지키려 만든 가드 테스트가 실제 오류를 잡아냈다. 챔피언스 아이템 목록과 루트
아이템 사전을 슬러그 기준으로 대조했더니, 메가스톤 다섯 개의 한국어명이 서로 달랐다.

```
alakazite: champions=후딘나이트  root=후디나이트
charizardite-x: champions=리자몽나이트 X  root=리자몽나이트X
```

원인은 챔피언스 아이템 생성기가 한국어명을 종족명+"나이트"로 독립 파생하면서, 받침 처리(후딘
→ 후디나이트)와 X·Y 앞 공백을 잘못 넣은 것이었다. 어느 쪽이 맞는지는 추측하지 않고 공식 표기를
찾아 확인했다 — 알라카잠 메가스톤의 정식명은 중복 받침을 떨군 "후디나이트"이고, X·Y는 공백
없이 붙인 "리자몽나이트X"가 맞았다.

여기서 1편의 원칙(데이터는 손으로 입력하지 않는다)과 부딪혔다. 그래서 데이터만 고치지 않고
**생성기를 먼저 고쳤다.** 메가도 루트 사전의 공식명을 우선 쓰고, 루트에 없는 챔피언스 오리지널
메가에만 파생값을 쓰되 공백 없이 붙이도록. 그렇게 생성기가 낼 값과 똑같이 데이터를 교정하고,
가드 테스트를 메가까지 확장해 다시 어긋나지 않게 못을 박았다.

## 그 외 정리

미사용 의존성을 들어냈고, SV·클립보드 시대에 작성돼 더는 맞지 않는 설계 문서들을 제거하면서
그 문서를 참조하던 곳도 함께 정리했다.

## 정리

리팩토링의 교훈은 "감을 증거로 바꾼 뒤 손대라"였다. 번들이 크다는 감은 측정(3.1MB→2.4MB)으로,
lint이 부실하다는 감은 설정 부재 확인으로, 데이터가 틀어졌다는 감은 가드 테스트로 확인했다.
그리고 데이터를 고칠 때는 항상 생성기부터 고쳤다. 다음 편은 이 위에 올린 커뮤니티·AI 확장
기능들이다.
