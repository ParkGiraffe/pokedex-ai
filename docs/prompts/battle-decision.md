# 프롬프트 가이드 — 실시간 배틀 의사결정

> `serializeForClaude("battle-decision")` paste를 받았을 때 Claude(대화형 AI)의 응답 표준.
> 어휘는 `docs/lexicon.md`를 따른다. 이모지 금지. 깊이 0(상대 응수 트리 없음) 기준.

## 입력 구조

- `## 현재 턴`·날씨·필드·트릭룸
- `## 내 파티`: 전체 슬롯
- `## 상대 공개분`: 종족명
- `## 내 액티브 기술 옵션`: 내 액티브 4기술의 데미지·KO 확률(16롤 중 N롤). **상대 0투자 중립 가정**. 이미 계산된 값이니 재계산하지 말 것.
- `## 요청`·`## 원본 데이터`

## 응답 구조

1. 옵션 Top 3 추천 (한국 어휘로 근거: 기술 vs 교체)
2. 각 옵션의 KO 확률·위험 요소
3. "이 옵션이 흔들리는 경우" (상대가 어떤 응수를 하면 추천이 바뀌는지)
4. 표준 JSON 코드블록 (`ClaudeResponseSchema`, task="battle-decision")

## 규칙

- KO 확률은 16롤 기준이다. "73%"를 확신처럼 단정하지 말고 "16롤 중 N롤" 감각으로 답한다.
- 상대 도구·테라·EV는 미상(0투자 가정)이다. "상대가 내구 투자면 확정타가 풀린다" 같은 가정 의존성을 명시한다.
- 교체 추천 시 들어올 픽과 그 픽의 상대 액티브 대비 우위를 근거로 댄다.
- 단정 금지. 사실(확정타/난수타)은 OK, "무조건 이긴다"는 금지.
- 메타 미수집 상태이므로 상대 행동 빈도 주장 대신 종족값·상성·스피드로 근거를 댄다.

## actionable 사용

- `kind`는 change-move(이 기술 선택) 또는 swap-slot(교체). `slot`·`from`·`to`·`reason` 채운다.
