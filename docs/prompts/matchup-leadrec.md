# 프롬프트 가이드 — 매치업 선두 추천

> `serializeForClaude("matchup-leadrec")` paste를 받았을 때 Claude(대화형 AI)의 응답 표준.
> 어휘는 `docs/lexicon.md`를 따른다. 이모지 금지.

## 입력 구조

- `## 내 파티`: 6슬롯 상세
- `## 상대 공개분`: 공개된 종족명 (부분 정보일 수 있음)
- `## 매치업 점수`: 내 픽별 verdict(유리/불리/호각)·커버리지·선두 점수. **참고용 결정론 계산값**(타입·스피드 기반, 기술셋 미반영). 재계산하지 말 것.
- `## 요청`·`## 원본 데이터`

## 응답 구조

1. 선두 후보 Top 3 우선순위 (한국 어휘로 근거)
2. 각 후보의 예상 상대 응수 (가능성 높은 순 2~3개)
3. "이 후보를 피해야 할 경우" 명시
4. 표준 JSON 코드블록 (`ClaudeResponseSchema`, task="matchup-leadrec")

## 규칙

- 점수는 참고용이다. 점수만 보고 단정하지 말고, 점수 + 메타 + 내 빌드 의도를 합쳐 추천한다.
- 매치업 점수는 기술셋·도구·테라를 모른다. 추천 시 "상대 도구/테라에 따라 바뀌는 지점"을 명시한다.
- 상대가 부분 공개면, 공개분 기준으로 답하고 "비공개 슬롯에 X가 있으면 추천이 바뀐다"를 적는다.
- 단정 금지. "막을 슬롯이 없다"(사실)는 OK, "확실히 진다"는 금지.
- 메타가 미수집이면 픽률 기반 주장 대신 종족값·상성·스피드로만 근거를 댄다.

## actionable 사용

- `kind`는 보통 swap-slot(다른 선두 제안). `slot`·`reason` 채운다. 선두 추천이라 change-* 는 드물게.
