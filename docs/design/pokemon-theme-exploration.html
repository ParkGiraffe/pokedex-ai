포켓몬스러운 색감 — 견본 제안

이 페이지는 견본일 뿐 앱에 적용되지 않았다. 현재 앱은 shadcn 기본값(채도 0 zinc 회색 + 임의 emerald)이라 "AI가 뽑은 색"처럼 보인다. 아래는 포켓몬 정통 아이덴티티에 기반한 3개 방향이다. 같은 미니 대시보드를 테마별로 렌더해 비교한다.

3. HP 바 — 포켓몬 시그니처 (초록 → 노랑 → 빨강)

배틀 화면의 상대 HP%는 단일 색이 아니라 잔량에 따라 색이 바뀌어야 한다. 게임 그대로의 신호.

HP100~50%
HP50~20%
HP20% 이하

4. 왜 이게 더 "포켓몬스러운가"

현재가 "AI스러운" 이유

  • 배경·카드·테두리가 전부 oklch(... 0 0) — 채도 0 순수 회색. v0/shadcn 기본 그대로.
  • 액센트가 emerald 하나. 포켓몬과 의미적 연결이 없다.
  • 타입·약점 같은 도메인 데이터까지 generic 초록/빨강으로 칠한다.
  • 브랜드 신호(레드·몬스터볼·도감 기기)가 0개.

제안의 공통 원리

  • 중립색에 미세한 색조(네이비/바이올렛/크림)를 입혀 "죽은 회색"을 제거.
  • 시그니처 1색을 포켓몬 정체성(몬스터볼·도감 레드 / SV 듀오)에서 가져온다.
  • 18타입 정식 색을 약점·배지·기술에 그대로 쓴다 = 가장 강한 도메인 신호.
  • HP는 초록→노랑→빨강, 골드는 테라스탈·강조에. 전부 게임에 근거.

적용은 보류 상태다. 방향(A/B/C 또는 조합)을 고르면 그때 apps/client/src/index.css의 @theme 토큰과 타입 색 유틸을 실제로 교체한다.
