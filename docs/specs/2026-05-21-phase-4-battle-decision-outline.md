# Phase 4 — 실시간 배틀 의사결정 AI (윤곽 spec)

> **상태:** 확정 (2026-05-21, Phase 3 완료 후 재설계).
> **재설계 결정 (§10 "깊이 0으로 후퇴" 채택):**
> - 메타 데이터가 비어 있어 상대 기술셋·EV·도구를 추정할 수 없다 → **27셀 게임 트리(상대 응수 포함)는 보류**. 대신 **깊이 0**: 내 액티브의 4기술별 데미지·KO 확률(16롤 중 KO 비율)만 결정론으로 계산한다.
> - 상대는 **0투자 중립 방어/HP**를 기본 가정(보수적·명시). 상대 HP%는 사용자가 입력. 정확 EV는 메타 수집 후 정교화.
> - 게임 트리·기댓값·다음 턴 매치업 재평가는 메타가 채워진 뒤의 후속 작업으로 미룬다(코드는 Phase 3 matchup 재활용 가능).
> - AI(Claude 대화형)가 KO 확률·상황을 받아 다음 턴 추천을 만든다. 앱은 게임 트리 탐색을 하지 않는다(아키텍처: AI=Claude).
> - `serializeForClaude("battle-decision")`에 내 옵션 표를 첨부한다.

## 1. 목표

배틀 도중의 현재 상황(필드·HP·랭크·날씨·필드·남은 멤버)을 입력으로 받아 **다음 턴 옵션별 데미지·확률·기댓값**과 **추천 행동(기술 vs 교체)**을 답한다.

이 Phase는 4 Phase 중 가장 어렵다 — **부분 정보·확률 분포·상대 행동 예측·다단계 결정**이 모두 결합. 윤곽 단계에서 "할 수 있는 만큼만 한다"는 보수적 접근을 명시.

## 2. 가정 (Phase 0~3 결과 기반)

- 매치업 점수 계산기가 안정 동작.
- 메타 데이터에 인기 세트·도구·테라가 충분히 들어 있음.
- 사용자가 "이 사이트가 배틀 보면서 옆에서 도움 주는 수준"을 어느 정도 기대하는지 합의됨 (Phase 3 마무리 시).

## 3. 범위 · 비범위

**범위:**
- "배틀 트래커" UI (Phase 1 `/battle` 또는 `/party`의 새 모드)
- 현 상황 입력 위젯 (HP %·랭크·도구 사용 여부·테라 사용 여부·남은 멤버·필드 효과)
- 부분 게임 트리 탐색 — 깊이 1턴 (내 선택 × 상대 가능 응수의 즉시 결과)
- 옵션별 데미지·KO 확률 산출
- 프롬프트 가이드 `docs/prompts/battle-decision.md`

**비범위:**
- 다중턴 미리 탐색 (예: 3턴 앞)
- 정확한 상대 EV 추정 (메타 평균 사용)
- 상대 도구 미사용 추적 (사용자가 수기로 입력)
- 배틀 로그 자동 파싱 (스크린샷 OCR·게임 통신 분석 X)

## 4. 게임 트리 탐색 윤곽

깊이 1 턴 (= 옵션 × 상대 옵션) 의 즉시 결과 평가:

```
나의 옵션 (각 슬롯별 4개 기술 + 5개 교체) = 최대 9개
상대 옵션 (메타 기반 Top 3 응수)         = 3개

= 27 (9×3) 결과 매트릭스. 각 결과에서:
  - 양쪽 HP 변동
  - 추가 효과 (랭크·상태이상·필드 갱신)
  - "이 턴 KO 발생?" 플래그
  - 다음 턴 시작 시점의 매치업 점수 (Phase 3 재활용)
```

각 셀의 데미지는 Phase 0의 `calculateDamage` 16롤을 그대로 사용. KO 확률은 16롤 중 KO 발생 비율.

다음 턴의 매치업 점수를 평가에 합쳐 **현재 선택의 기댓값**을 산출:

```typescript
expectedValue(myOption) = Σ (상대 응수 확률 × 다음 턴 매치업 점수)
```

## 6. 부분 정보 처리

- 상대 도구·테라타입을 모를 때: 메타 기반 가장 흔한 세트로 가정. 응답에 가정 명시.
- 상대 EV: Smogon 평균 EV 또는 가장 흔한 스프레드 사용.
- 상대 남은 파티가 비공개일 때: Top 30 메타에서 추정. 표시는 "추정값" 강조.

## 7. 프롬프트 가이드 (`docs/prompts/battle-decision.md`)

응답 구조:
1. 옵션 Top 3 추천 (한국 어휘로 근거)
2. 각 옵션의 KO 확률·다음 턴 매치업 변동·위험 요소
3. "이 옵션이 무서운 경우" (= 상대가 어떤 응수를 하면 추천이 흔들리는지)
4. 응답 마지막 표준 JSON

## 8. 위험 · 미해결 항목

| 항목 | 다룰 시점 |
|---|---|
| 27개 결과 매트릭스 계산 비용 (캐싱 없으면 매 paste마다 재계산) | Phase 4 시작 시점 — 캐싱·메모이제이션 설계 |
| 상대 응수 확률 모델 (메타 빈도만으로 충분한가?) | Phase 4 실측 후 결정. Top 3 균등도 OK일 수 있음 |
| "확률 환상" — 사용자가 KO 73%를 73% 확신으로 오해 | UI에 "16롤 중 N롤" 명시. 응답에서도 절대값 강조 안 함 |
| 다음 턴 매치업 점수의 정확도 (Phase 3의 모델이 깊이 1턴에 적합한가?) | Phase 3 점수가 다음 턴 매치업에도 의미 있는지 검증 후 결정 |
| 부분 정보 가정이 깨졌을 때의 재추천 (상대가 갑자기 테라 사용) | 사용자가 입력 갱신하고 다시 paste — UI 흐름 단순화 |

## 9. 완료 조건

- 배틀 트래커 UI에서 현 상황 입력 가능
- "지금 뭐 할까?" 클릭 → paste 데이터에 27개 옵션 매트릭스 포함
- 실제 배틀 10턴을 손으로 입력하면서 Claude 추천 받고, 본인 체감 추천 품질 만족
- KO 확률·매치업 점수가 결정론적으로 동일 입력 → 동일 출력
- `docs/prompts/battle-decision.md` 작성

## 10. 유연성 노트 (이 Phase는 특히 갈아엎힐 가능성 높음)

- **27개 매트릭스가 너무 무겁다고 판단되면** → 옵션 축소 (예: 내 선택만 9개 보여주고 상대 응수 비교는 응답 본문으로) 또는 깊이 0으로 후퇴
- **사용자가 배틀 중에 입력 시간 자체가 부담스러우면** → UI 단순화. 핵심 인풋(상대 픽·HP·랭크)만 입력하고 나머지는 가정
- **다음 턴 매치업 점수가 의미 없다고 검증되면** → 점수 빼고 단순 KO 확률만 표시
- **메타 기반 상대 응수 모델이 부정확하면** → 사용자가 수기로 "상대가 다음에 이 기술을 쓸 확률 60%" 같이 직접 입력하는 모드 추가
- **이 Phase 자체를 포기해도 됨** — Phase 0~3 만으로도 충분히 가치 있는 도구. 4 Phase는 본인이 배틀 중에 도움이 정말 되는지 검증한 뒤 진입
